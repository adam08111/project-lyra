Step 3 — Class D, enforced not assumed. - Every field renders through React default escaping. dangerouslySetInnerHTML and any innerHTML are FORBIDDEN in src/teacher/ — add a test that greps the directory for both and fails on a hit. - Treat payload jsonb as hostile at any depth; render only whitelisted fields the Step-0 inventory names, as text. - URLs in any field render as plain text (no anchor) in v1. - No CSV/XLSX export in v1 (formula-injection handling is the documented reason — Class D item 2 — deferred with a note in the § entry). - Characterization test: seed a hostile string (<img src=x onerror=alert(1)> and a javascript: URL) into a mock event's rule/topic/payload; render with the §104 happy-dom harness; assert the DOM contains the escaped text and no img/script node materialized.
